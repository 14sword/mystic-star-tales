# 项目代码审查与优化建议

对指定项目进行全面代码审查，输出结构化报告。

## 输入

- `$ARGUMENTS`：项目路径（默认当前目录）

## 工作流程

### 1. 项目概况

- 列出目录结构（`ls -la` + `find` 限深度 3）
- 识别技术栈（框架、语言、构建工具）
- 读取 `package.json` / `README.md` / 配置文件

### 2. 代码质量扫描

对每个主要源文件：

- **语法检查**：对 JS/TS 文件用 `node --check`，对 HTML 中的 `<script>` 用 Python 正则提取后检查
- **常见 bug 模式**：
  - 未声明变量、重复声明
  - 未使用的 import / 变量
  - 硬编码的 URL / 密钥
  - 缺少错误处理（无 try-catch、无 catch）
  - 内存泄漏风险（未清理的 setInterval / addEventListener）
  - CSS 覆盖冲突

### 3. 依赖检查

- 检查 `node_modules` 是否存在且完整
- 检查是否有已知漏洞（`npm audit` 如果可行）
- 检查 lock 文件是否与 package.json 一致

### 4. 输出格式

```markdown
# 项目审查报告：{项目名}

## 概况
- 技术栈：...
- 文件数：...
- 主要入口：...

## 发现的问题（按严重性排序）

### 🔴 严重
- [文件:行号] 问题描述 + 修复建议

### 🟡 中等
- [文件:行号] 问题描述 + 修复建议

### 🟢 建议
- [文件:行号] 优化建议

## 优化建议
1. ...
2. ...

## 总结
- 严重问题：N 个
- 中等问题：N 个
- 优化建议：N 个
```

## 注意事项

- 只读审查，不自动修改代码
- 如果用户指定了路径，使用 `$ARGUMENTS`；否则使用当前工作目录
- 对大型文件（>500行）重点审查开头和关键函数
- 中文输出报告
